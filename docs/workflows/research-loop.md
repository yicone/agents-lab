# Research Loop

Use this workflow when analyzing products, papers, announcements, workflows, or agent patterns for this repository.

## 1. Gather

- Collect only verifiable evidence
- Prefer primary sources when possible
- Capture concrete facts, dates, and scope

## 2. Classify

Assign the item to the project taxonomy:

- `axis/coding-agent`
- `axis/orchestrator`
- `axis/memory`
- `axis/skills`
- `axis/platform`
- `axis/prompts`

Add secondary labels only when useful, such as `product/*`, `topic/*`, `workflow/*`.

## 3. Evaluate

Use the project rubric:

- relevance
- adoptability
- differentiation
- evidence strength
- risk or cost

Default decisions:

- `deep-dive`
- `evaluate`
- `monitor`

## 4. Distill

Write back only durable conclusions, such as:

- stable preferences
- framework changes
- notable tradeoffs
- reproducible lessons
- product deltas that matter for future comparison

## 5. Persist

When storing reusable output, include:

- `proj/agents-lab`
- at least one `axis/*` tag
- one or more of `decision/*`, `workflow/*`, `topic/*`, or `product/*`

Keep entries short, searchable, and explicit about why the item matters.
